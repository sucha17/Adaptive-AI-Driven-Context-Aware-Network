# 🌐 Adaptive AI-Driven Context-Aware Network
โครงงานนี้คือสถาปัตยกรรมเครือข่ายแห่งอนาคตที่นำ **ปัญญาประดิษฐ์ (AI)** มาเป็นสมองส่วนกลางในการบริหารจัดการเครือข่าย เพื่อก้าวข้ามข้อจำกัดของเครือข่ายแบบดั้งเดิมที่มักจะมีการตั้งค่าแบบตายตัว (Static)

link project: https://adaptive-ai-driven-context-aware-network-u68xti6263saszscygnug.streamlit.app/
คือ SDN Dashboard ทำหน้าที่เป็น Application Plane ของระบบครับ โดยตัว Dashboard นี้จะคอย Monitor ข้อมูลและ Generate ชุดคำสั่ง API เพื่อควบคุมเครือข่ายแบบอัตโนมัติ 

## 📌 โครงการนี้คืออะไร?
Adaptive AI-Driven Context-Aware Network คือสถาปัตยกรรมเครือข่ายรูปแบบใหม่ที่ใช้ปัญญาประดิษฐ์ (AI) เป็นแกนหลัก ซึ่งต่างจากเครือข่ายแบบดั้งเดิมที่ตั้งค่าตายตัว เครือข่ายนี้สามารถรับรู้บริบท (พฤติกรรมผู้ใช้, สถานที่, สภาพเครือข่าย) และปรับตัวอัตโนมัติแบบเรียลไทม์ เพื่อให้ผู้ใช้งานได้รับประสบการณ์ที่ดีที่สุดและเครือข่ายทำงานอย่างมีประสิทธิภาพสูงสุด


## 🎯 วัตถุประสงค์ทางการศึกษา (Educational Objectives)
- **ประยุกต์ใช้แนวคิดคอมพิวเตอร์ที่รับรู้บริบทกับระบบเครือข่าย**
- **ออกแบบการจัดสรรทรัพยากรและการหาเส้นทางที่ขับเคลื่อนด้วย AI**
- **พัฒนาแบบจำลองเครือข่ายที่รองรับ Use Cases จริง** (Smart Campus, Smart Stadium, Emergency Response)
- **จัดการข้อจำกัดทางเทคโนโลยี** เช่น ความเป็นส่วนตัว, ปัญหาคอขวดในการประมวลผลของ AI, และการรองรับอุปกรณ์รุ่นเก่า
- **ฝึกหัดการเขียนเอกสารสถาปัตยกรรมซอฟต์แวร์อย่างเป็นระบบ**


## 💡 ขอบเขตและข้อจำกัด (Scope and Constraints)

เพื่อให้การดำเนินโครงงานเป็นไปตามเป้าหมายและระยะเวลาที่กำหนด จึงได้กำหนดขอบเขตของการศึกษาและพัฒนาไว้ดังนี้:
**✅ สิ่งที่อยู่ภายในขอบเขตโครงงาน (In-Scope)**
* **ด้านสถาปัตยกรรม:** มุ่งเน้นการออกแบบและพัฒนาสถาปัตยกรรมเครือข่ายภายใต้ 3 แกนหลัก ได้แก่ เครือข่ายที่รู้จักตนเอง (Self-Aware), ปรับตัวอัตโนมัติ (Self-Adaptive) และยึดผู้ใช้เป็นศูนย์กลาง (User-Centric)
* **ด้านการสร้างแบบจำลอง:** พัฒนาและทดสอบการทำงานของระบบผ่าน **แบบจำลองซอฟต์แวร์ (Software Simulation)** เป็นหลัก
* **ด้านความปลอดภัย:** ครอบคลุมเฉพาะการพัฒนาระบบปกปิดข้อมูลส่วนบุคคล (Anonymization) และการสร้างระบบสำรอง (Fail-Safe/Manual Override) เพื่อป้องกันกรณีที่ AI ตัดสินใจผิดพลาด
* **ด้านการทดสอบ:** จำลองสถานการณ์และทดสอบการทำงานใน 3 กรณีศึกษาหลัก ได้แก่ พื้นที่การศึกษา (Smart Campus), พื้นที่ที่มีความหนาแน่นสูง (Smart Stadium) และสถานการณ์ฉุกเฉินระดับท้องถิ่น (Emergency Response)


## 📅 แผนการดำเนินงานเบื้องต้น (High-Level Implementation Plan)
โครงการนี้ใช้รูปแบบการทำงานแบบ Agile โดยแบ่งการพัฒนาออกเป็น 4 ระยะ (4-Week Sprint):
- **Sprint 1: Architecture & Infrastructure Setup (สัปดาห์ที่ 1)** - วางโครงสร้างสถาปัตยกรรมเครือข่าย ออกแบบโครงสร้างข้อมูลบริบท และเตรียมสภาพแวดล้อมแบบจำลอง (Simulation)
- **Sprint 2: Core Protocols & AI Routing (สัปดาห์ที่ 2)** - พัฒนาอัลกอริทึมหาเส้นทางด้วย AI และลอจิกการจัดการทราฟฟิกอัตโนมัติ
- **Sprint 3: Privacy, Fail-Safe & Integration (สัปดาห์ที่ 3)** - บูรณาการระบบรักษาความเป็นส่วนตัว ตั้งกฎโหมดฉุกเฉิน (Override) และรวมทุกระบบเข้าด้วยกัน
- **Sprint 4: Final Testing & Documentation (สัปดาห์ที่ 4)** - รันการทดสอบระบบเต็มรูปแบบตาม Use Cases จำลองการทำงาน และสรุปเอกสารเพื่อนำเสนอ

📘 คู่มือการใช้งานระบบ
ระบบนี้ถูกออกแบบมาเพื่อทำหน้าที่เป็น Application Plane ในสถาปัตยกรรม Software-Defined Networking (SDN) โดยใช้หลักการ Closed-loop Automation ในการบริหารจัดการเครือข่ายอัตโนมัติ

1. การเลือกสถานการณ์ (Scenario Selection)
วิธีใช้: ที่แถบด้านซ้าย (Sidebar) ให้เลือกเมนู "เลือกสถานการณ์ (Scenario)"

สิ่งที่เกิดขึ้น: ระบบจะเปลี่ยน Logic การตัดสินใจของ AI ทันที

Smart Campus: โหมดปกติ เน้นการประหยัดทรัพยากร

Smart Stadium: โหมดรองรับผู้ใช้จำนวนมาก AI จะขยาย Bandwidth เพื่อลด Latency

Emergency: โหมดวิกฤต AI จะตัด Bandwidth ส่วนเกินและเพิ่ม Priority (50000) ให้กับหน่วยกู้ภัยโดยเฉพาะ

2. การตรวจสอบชุดคำสั่ง API (Northbound API Analysis)
ตำแหน่ง: กลางหน้าจอในส่วน "Generated REST API Payload"

มื่อเปลี่ยน Scenario ให้สังเกตว่าค่าในกล่อง JSON จะเปลี่ยนตามแบบ Real-time

ค่า priority จะเปลี่ยนตามความสำคัญของงาน

ค่า action จะเปลี่ยนเป็นคำสั่งควบคุม (เช่น RESERVE_MIN_BANDWIDTH)

คือ ชุดคำสั่งมาตรฐาน RESTCONF ที่ระบบสร้างขึ้นเพื่อส่งไปหา SDN Controller (ONOS) เพื่อแก้ Flow Table ใน Switch จริง

3. การวิเคราะห์ข้อมูลจาก Log จริง (Real-world Log Capture)
ตำแหน่ง: แถบด้านซ้ายล่างสุด หัวข้อ "Real-world Log Capture"

ข้อมูลในกล่องนี้ไม่ได้มาจากการสุ่ม แต่เป็นการดึงค่าจากไฟล์ network_logs.json ในระบบ

เพื่อโชว์ว่าระบบรองรับการทำ Data Parsing จาก Flow Statistics ของ Switch จริง เพื่อนำมาใช้ประมวลผลร่วมกับ AI

4. ระบบรักษาความเป็นส่วนตัว (Privacy Toggle)
วิธีใช้: กดเปิดปุ่ม "Enable Data Masking (Anonymization)"

สิ่งที่เกิดขึ้น: ระบบจะจำลองการทำ Data Anonymization เพื่อปกปิดตัวตนของผู้ใช้งานในเครือข่ายก่อนส่งข้อมูลไปวิเคราะห์ที่ Cloud
